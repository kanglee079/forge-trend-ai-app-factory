# Learning Memory

ForgeTrend does not train a model. It stores practical run memory:

- run evaluations
- failure patterns
- learning rules
- provider success
- archetype quality scores

Learning rules are transparent and can be shown in `/learning`.

Examples:

- If Codex auth is missing, fallback deterministic and notify the user.
- If generic copy appears, force deeper blueprint and interactive core flow next run.
- If Vietnamese education apps pass often, prefer the education archetype.
