# App Generation Quality Standard

A generated app must be product-specific, not a generic shell.

Required app surfaces:

- Onboarding
- Home dashboard
- Core feature flow
- Progress/history
- Settings
- Privacy/about
- Help/FAQ or guidance
- Paywall when monetization is enabled
- Loading, empty, error, success, snackbar, and confirmation states
- Dark mode
- Vietnamese default when `target_language=vi`
- English fallback

Required artifacts:

- `app_blueprint.json`
- `app_blueprint.md`
- `product_score_report.json`
- `product_score_report.vi.md`
- `quality_gate_report.md`
- `store_readiness_report.md`
- `store_assets/*`

Blocked phrases in generated visible Dart copy include generic template wording such as "replace this", "sample error state", "local sample data", "adapt this item", "placeholder included", "generated by forgetrend", "core flow", "next action", and "review the first workflow".
